# Standards

This folder contains standards, principles, best practices, and implementation guidance for the Customer API-led Reference Architecture. These documents promote consistency across API specifications, Mule implementations, and related assets.

Examples include:

- Naming conventions
- Error handling standards
- API versioning guidance
- Security recommendations
- Logging standards
- Design principles
- Implementation best practices

## Available Standards

### guide-error-code-taxonomy.md

This document defines the standard error code taxonomy for the reference architecture, supporting consistent and machine-readable error responses across APIs.
